Inheritance example:


class Bicycle  
{ 

// the Bicycle class has two fields 

    public int gear; 
    public int speed;    

// the Bicycle class has one constructor 

    public Bicycle(int gear, int speed) 
    { 
        this.gear = gear; 
        this.speed = speed; 
    }        

// the Bicycle class has three methods 

    public void applyBrake(int decrement) 
    { 
        speed -= decrement; 
    }           
    public void speedUp(int increment) 
    { 
        speed += increment; 
    }       

// toString() method to print info of Bicycle 

    public String toString()  
    { 
        return("No of gears are "+gear 
            +"\n"
            + "speed of bicycle is "+speed); 
    }  
}  

// derived class 

class MountainBike extends Bicycle  
{        

// the MountainBike subclass adds one more field 

    public int seatHeight; 

// the MountainBike subclass has one constructor 

    public MountainBike(int gear,int speed, int startHeight) 
    { 

// invoking base-class(Bicycle) constructor 

        super(gear, speed); 
        seatHeight = startHeight; 
    }    

// the MountainBike subclass adds one more method 

    public void setHeight(int newValue) 
    { 
        seatHeight = newValue; 
    } 

// overriding toString() method 

// of Bicycle to print more info 

    @Override public String toString() 
    { 
        return (super.toString()+ 
            "\nseat height is "+seatHeight); 
    } 
          
}

Relational and Logical operators:



Relational operators:
int num = 5;

// Equal to
    num == 5; // true

// Not equal to
    num != 5; // false

// Greater than OR equal to
    num >= 5; // true

// Less than OR equal to
    num <= 5; // true

// Greater than
    num > 4; // true

// Less than
    num < 4; // false

Logical operators:

// Reverses the result, returns false if the result is true
// !
if !(num == 5)
    returns false; 

// Returns true if both statements are true
// &&
if (num <= 5 &&  num < 10)
    returns true;
    
// Returns true if one of the statements is true
// ||
if(num < 5 || num < 4)
    return false;

Switch vs if else


                        
    

    Deciding when to use which is paramount. Here are some reasons to use either:
    


  1. Check the Testing Expression: An if-then-else statement can test expressions based on ranges of values or conditions, whereas a switch statement 
tests expressions based only on a single integer, enumerated value, or String object.
    2. 

  2. 
Switch better for Multi way branching: When compiler compiles a switch statement, it will inspect 
each of the case constants and create a “jump table” that it will use for selecting the path of execution depending on the value of the expression. 
Therefore, if we need to select among a large group of values, a switch statement will run much faster than the equivalent logic coded using a sequence 
of if-elses. The compiler can do this because it knows that the case constants are all the same type and simply must be compared for equality 
with the switch expression, while in case of if expressions, 
the compiler has no such knowledge.

    3. 

  3. 
if-else better for boolean values: 
If-else conditional branches are great for variable conditions that result into a boolean, 
whereas switch statements are great for fixed data values.

    4. 

  4. 
Clarity in readability: A switch looks much cleaner when you have to combine cases. Ifs  are quite vulnerable to errors too. 
Missing an else statement can land you up in havoc. 
Adding/removing labels is also easier with a switch and makes your code significantly easier to change and maintain.

Differences


                        
    

  1. For: 
Initialization may be either in loop statement or outside the loop.

While:
Initialization is always outside the loop.

    2. 

  2. For:
Once the statement(s) is executed then after increment is done.

While:
can be done before or after the execution of the statement(s).

    3. 

  3. For:
It is normally used when the number of iterations is known.

While:
It is normally used when the number of iterations is unknown.
